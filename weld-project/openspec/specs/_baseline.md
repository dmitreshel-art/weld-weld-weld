# Baseline Analysis — Weld-Weld-Weld

> Code analysis from 2026-03-03

## Executive Summary

**Project Status: CRITICAL - NON-FUNCTIONAL**

The calculator cannot perform any calculations due to missing input fields and unconnected variables. The application appears to be a work-in-progress with significant architectural issues.

---

## Critical Issues Found

### CRIT-001: Missing Input Fields (Severity: 10/10)

**Location:** `index.html`
**Problem:** Variables required for formulas don't have corresponding UI inputs.

| Variable | Description | HTML Element | Status |
|----------|-------------|--------------|--------|
| `thickness` | Толщина металла | `#thicknessList` | ⚠️ Select, no input |
| `gap` | Зазор (b) | — | ❌ MISSING |
| `blunting` | Притупление (c) | — | ❌ MISSING |
| `weldWidth` | Ширина шва (e) | — | ❌ MISSING |
| `gainHeight` | Высота усиления (g) | — | ❌ MISSING |
| `angleAlpha` | Угол разделки α | `#cuttingAngle` | ⚠️ Only text display |
| `angleBetta` | Угол разделки β | — | ❌ MISSING |
| `radiusTransition` | Радиус перехода (R) | — | ❌ MISSING |
| `diameter` | Диаметр трубы (d) | — | ❌ MISSING |
| `refractiveHeight` | Высота преломления | — | ❌ MISSING |
| `protrusionEnd` | Выступ конца | — | ❌ MISSING |

| `gainHeightSecond` | Высота усиления 2 (g1) | — | ❌ MISSING |

**Impact:** ALL calculations fail with `NaN` or `undefined`.

### CRIT-002: Variables Not Connected (Severity: 10/10)

**Location:** `scripts/formulas.js:1-30`
**Problem:** Variables declared but never assigned values from UI.

```javascript
// These are NEVER assigned:
let gap;                    // undefined
let blunting;               // undefined
let weldWidth;              // undefined
let gainHeight;             // undefined
let angleAlpha;             // undefined
let angleBetta;             // undefined
let radiusTransition;       // undefined
let diameter;               // undefined
let refractiveHeight;       // undefined
let protrusionEnd;          // undefined
let distanceBetweenEdgesCitcularSelection;  // undefined
let weldWidthSecond;        // undefined
let gainHeightSecond;       // undefined

// Only this gets a value (but only on page load, not on change):
let thickness = thicknessCount.value;  // Gets initial value only
```

**Impact:** `crossSectionalArea` always = `NaN`

---

### CRIT-003: exel.json Not Used (Severity: 8/10)

**Location:** `exel.json`, `scripts/calculation.js`
**Problem:** JSON data file exists but is never loaded.

```javascript
// exel.json:
[{"connectionType": "С1", "thickness": "0.5", "gap": "0.5", ...}]

// calculation.js - data is HARDCODED instead:
let typeOfConnectionArray5264 = ["", "С1", "С28", "С3", "С2", ...];
```

**Impact:** 
- Cannot update data without code changes
- Limited to hardcoded values
- exel.json maintenance waste

---

### CRIT-004: Thickness Limited to Select (Severity: 6/10)

**Location:** `index.html`, `scripts/calculation.js`
**Problem:** Thickness is a `<select>` with hardcoded options, not free input.

```html
<select id="thicknessList" class="userform__select userform__select-d">
  <!-- Options generated by JS, not from exel.json -->
</select>
```

**Impact:**
- User cannot enter arbitrary thickness
- Limited to predefined values
- Inconsistent with exel.json data

---

### CRIT-005: No Error Handling (Severity: 7/10)

**Location:** `scripts/formulas.js`
**Problem:** No try-catch, no validation, no error messages.

```javascript
// This will silently fail:
crossSectionalArea = thickness * gap + 0.75 * (weldWidth * gainHeight);
// = undefined * undefined + 0.75 * (undefined * undefined)
// = NaN
```

**Impact:** User sees no feedback, calculator appears broken.
