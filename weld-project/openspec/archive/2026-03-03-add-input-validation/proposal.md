# add-input-validation

## Intent

Добавить валидацию вводимых параметров: проверка min/max значений, типы данных, обязательные поля.

Сейчас параметры принимают любые значения, включая отрицательные и некорректные. Это может привести к ошибкам расчёта или некорректным результатам.

## Scope

### In Scope

- [ ] Валидация числовых значений (положительные числа)
- [ ] Проверка min/max для каждого параметра по типу соединения
- [ ] Отображение ошибок валидации пользователю
- [ ] Блокировка расчёта при невалидных данных
- [ ] Очистка полей при смене типа соединения

### Out of Scope

- ❌ Серверная валидация (только клиентская)
- ❌ Валидация thickness select (уже есть список)
- ❌ Международзация сообщений об ошибках
- ❌ Redesign UI для ошибок

## Success Criteria

1. **Given** пользователь ввёл отрицательное значение
   **When** нажимает ПОДСЧИТАТЬ
   **Then** показывается ошибка "Значение должно быть положительным"

2. **Given** пользователь ввёл значение > max для типа
   **When** нажимает ПОДСЧИТАТЬ
   **Then** показывается ошибка "Значение должно быть не более X"

3. **Given** все поля валидны
   **When** нажимает ПОДСЧИТАТЬ
   **Then** расчёт выполняется без ошибок

4. **Given** пользователь сменил тип соединения
   **When** новый тип выбран
   **Then** поля сбрасываются на defaults
