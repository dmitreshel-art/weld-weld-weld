# Change Proposal: Desktop Form Layout Optimization

## Status
🟢 APPROVED — Вариант B (результаты справа)

## Summary
Оптимизация расположения полей формы для уменьшения прокрутки на десктопе. Все поля должны быть видны без скрола при загрузке страницы.

## Problem Statement

### Текущие проблемы (Desktop):
1. **Поля невидимы при загрузке** — форма начинается ниже экрана, пользователь не видит поля
2. **Вертикальный layout** — все поля в один столбец, требует прокрутки
3. **Избыточный вертикальный скрол** — около 20 полей в один ряд
4. **Плохой UX** — результаты расчёта скрыты внизу

### Текущая структура формы:
```
[Способ сварки]      ← 1 поле
[НТД по сварке]      ← 1 поле  
[Вид соединения]     ← 1 поле
[Толщина]            ← 1 поле
[Угол разделки]      ← 1 поле
━━━━ Параметры шва ━━━━
[Зазор] [Притупление] [Ширина шва] [Высота усиления] [Угол α] [Угол β] [Радиус] [Катет] [Диаметр]
[Доп. параметры] (скрыты)
[КНОПКА]
━━━━ Результаты ━━━━
[Fn] [Mn] [Ge] [He]
```

---

## Requirements

### Функциональные требования:

**REQ-001: Видимость без скрола**
- При загрузке страницы все основные поля должны быть видны
- Допускается скрол только для результатов расчёта (если они скрыты)

**REQ-002: Компактный layout для Desktop**
- Разрешение target: 1920x1080 и 1366x768
- Минимум полей виден: все 4 селекта + параметра шва
- Группировка связанных полей

**REQ-003: Сохранение структуры на Mobile**
- Мобильная версия остаётся без изменений (вертикальный layout)
- Изменения только для desktop (min-width: 769px)

### UI требования:

**UI-001: Grid-Layout для Desktop**
- Основные селекты: 2 колонки (2x2)
- Параметры шва: 3 колонки (3x3)
- Кнопка: полная ширина или по центру

**UI-002: Компактные подписи**
- Уменьшить размер label (с 20px до 14-16px)
- uppercase оставить для семантики

**UI-003: Результаты расчёта**
- Опционально: вывести справа или сверху

## Proposed Solution

### Вариант A: Двухколоночный grid

```
┌─────────────────────────────────────────────────────────────┐
│  [Способ сварки]     │  [НТД по сварке]                      │
│  [Вид соединения]    │  [Толщина]                            │
├─────────────────────────────────────────────────────────────┤
│  УГОЛ РАЗДЕЛКИ: ___                                               │
├─────────────────────────────────────────────────────────────┤
│  ПАРАМЕТРЫ ШВА                                                      │
│  [Зазор] [Притупление] [Ширина] [Высота] [Угол α] [Угол β]...
│  [Радиус] [Катет] [Диаметр]                                        │
├─────────────────────────────────────────────────────────────┤
│  [          ПОДСЧИТАТЬ          ]                                  │
└─────────────────────────────────────────────────────────────┘
```

### Вариант B: Компактные поля с результатами справа

```
┌──────────────────────────┬─────────────────────┐
│  [Способ сварки]         │  РЕЗУЛЬТАТЫ         │
│  [НТД по сварке]         │  Fn: ___ мм²       │
│  [Вид соединения]        │  Mn: ___ г/м       │
│  [Толщина]               │  Ge: ___ г/м       │
│  ─────────────           │  He: ___ г         │
│  [Угол разделки]         │                    │

│  [b][c][e][g][α][β][R][k][d]         │

│  [    ПОДСЧИТАТЬ    ]    │                    │
└──────────────────────────┴─────────────────────┘
```

## Scope

### In Scope:
- Изменение CSS для desktop-версии (grid-layout)
- Перестановка HTML структуры (группировка полей)
- Оптимизация высоты формы

### Out of Scope:
- Мобильная версия (остаётся вертикальной)
- Изменение логики расчётов
- Изменение визуального стиля (цвета, шрифты)
- Дополнительные поля

## Acceptance Criteria

| ID | Критерий | Как проверить |
|----|----------|---------------|
| AC-001 | На 1920x1080 форма полностью видна без скрола | Открыть страницу, проверить viewport |
| AC-002 | На 1366x768 основные поля видны, минимальный скрол | Открыть страницу |
| AC-003 | Основные select-поля в 2 колонки | Визуальная проверка |
| AC-004 | Параметры шва в 3 колонки | Визуальная проверка |
| AC-005 | Мобильная версия без изменений | Проверить на 375px |
| AC-006 | Функциональность сохранена | Пройти весь flow расчёта |
