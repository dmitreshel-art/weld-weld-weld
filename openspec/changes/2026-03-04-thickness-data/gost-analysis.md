## Выводы

### 1. gost-data.js — наиболее полный источник
- **18 типов соединений** для ГОСТ 5264-80
- Структурированные данные с параметрами (b, e, g, R, k, etc.)
- Диапазоны толщин указаны точно по ГОСТ

### 2. exel-gost.json — частичный источник
- **10 типов соединений**
- Данные совпадают с gost-data.js для общих типов
- Отсутствуют: С4, С5, С6, С7, С28, У4, У5, У6, Т3

### 3. calculation.js (hardcoded) — ограниченный
- **7 типов** для MMA/MIG-MAG
- Содержит С28, которого нет в gost-data.js
- Не использует данные из gost-data.js

### 4. data/mig-mag.json — неполный
- Только С1 с полными данными для МП
- Остальные типы пустые

## Проблема

**Текущая реализация:**
- `calculation.js` имеет hardcoded массивы для dropdown
- `gost-data.js` существует, но **не используется**
- Дублирование данных в разных форматах

**Последствия:**
- Пользователь видит 40+ типов в dropdown
- Только 7 типов работают (hardcoded)
- 18 типов есть в gost-data.js, но не подключены
