# Change Proposal: Missing Thickness Data for Connection Types

## Status
🔵 PROPOSED — Awaiting discussion

## Summary
Множество типов соединений в выпадающем списке не имеют обработчиков для выбора толщины. Пользователь может выбрать тип соединения (например, С9), но поле "Толщина" остаётся пустым — нет вариантов для выбора.

---

## Problem Statement

### Текущее поведение:
1. Пользователь выбирает "Способ сварки" (MMA)
2. Выбирает "НТД" (ГОСТ 5264)
3. Выбирает "Вид соединения" (например, С9)
4. Поле "Толщина" остаётся пустым — **нет вариантов**

### Техническая причина:
В `scripts/calculation.js` обработчик `typeOfConnection.addEventListener("change", ...)` реализован только для части типов соединений:

| Тип | Реализовано | Толщины |
|-----|-------------|---------|
| С1, С3 | ✅ | 1-4 мм |
| С28 | ✅ | 1-12 мм |
| С4, С5, С6 | ✅ | 1-4 мм |
| С7 | ✅ | 2-5 мм |
| **С2, С8, С9, С10-С27, С39-С45** | ❌ | — |
| **У1-У10** | ❌ | — |
| **Т1-Т9** | ❌ | — |
| **Н1, Н2** | ❌ | — |

### Доступные в списке, но без данных:
```
С2, С8, С9, С10, С11, С12, С13, С14, С15, С16, С17, С18, С19,
С20, С21, С23, С24, С25, С26, С27, С39, С40, С42, С43, С45,
У1, У2, У4(1), У4(2), У5(1), У5(2), У6, У7, У8, У9, У10,
Т1, Т3, Т5, Т6, Т7, Т8, Т9,
Н1, Н2
```

---

## Options Analysis

### Вариант A: Добавить недостающие данные

**Подход:** Найти данные по толщинам для всех типов в ГОСТ и добавить обработчики.

**Плюсы:**
- Полная функциональность
- Пользователь может использовать любой тип соединения

**Минусы:**
- Требует доступ к ГОСТ документам
- Значительный объём работы
- Риск неточностей без верификации

**Необходимо:**
- ГОСТ 5264-80 (РД)
- ГОСТ 14771-76 (МП)
- ГОСТ 16037-70 (РАД)
- ГОСТ 8713-79 (АФ)

**Оценка времени:** 2-4 часа при наличии документов

---

### Вариант B: Ограничить список рабочими типами

**Подход:** Оставить в dropdown только те типы, для которых есть данные.

**Плюсы:**
- Быстрое решение
- Нет ложных ожиданий у пользователя
- Минимальный риск

**Минусы:**
- Ограниченная функциональность
- Пользователь не может выбрать некоторые типы

**Изменения:**
- Удалить нереализованные типы из `typeOfConnectionArray5264`
- Аналогично для других ГОСТ

**Оценка времени:** 30 минут

---

### Вариант C: Ручной ввод толщины

**Подход:** Добавить fallback — если нет预设анных толщин, показать поле для ручного ввода.

**Плюсы:**
- Гибкость для пользователя
- Не теряется функциональность

**Минусы:**
- Пользователь должен знать толщину самостоятельно
- Нет валидации по ГОСТ
- Более сложная логика

**Изменения:**
- Добавить условие: если тип без данных → показать input вместо select
- Или всегда показывать input как альтернативу

**Оценка времени:** 1-2 часа

---

### Вариант D: Частичное покрытие (приоритетные типы)

**Подход:** Добавить данные только для часто используемых типов (например, С8, С17).

**Плюсы:**
- Баланс между усилиями и пользой
- Можно расширять постепенно

**Минусы:**
- Неполное решение
- Требует анализ популярности типов

**Оценка времени:** 1 час на 5-10 типов

---

## Recommendation

**Рекомендую начать с Варианта B (ограничение списка)** как минимальное решение:
- Убирает путаницу для пользователя
- Быстро реализуется
- Можно расширить позже (Вариант A)
